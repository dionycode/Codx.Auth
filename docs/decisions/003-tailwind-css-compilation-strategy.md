# ADR-003: Compile Tailwind CSS to a Static Asset; Use CDN Play in Development Only

**Status**: Proposed  
**Date**: 2026-05-01

## Decision

The Tailwind CSS for the Tailwind theme is delivered as a pre-compiled static file at `wwwroot/themes/tailwind/tailwind.css`, generated by the Tailwind CLI (`npx tailwindcss`). During local development, the Tailwind CDN Play script (`<script src="https://cdn.tailwindcss.com">`) may be used as a convenience. The CDN script must never be deployed to production.

## Context

Tailwind CSS v4 generates CSS by scanning source files for class names and emitting only the utilities actually used. Two delivery options exist:

**Option A — Compiled static asset:**  
Run `npx tailwindcss -i input.css -o wwwroot/themes/tailwind/tailwind.css --minify` as part of CI. The output file is committed or built on demand.

**Option B — CDN Play (dev convenience):**  
`<script src="https://cdn.tailwindcss.com">` injects the full Tailwind runtime in the browser. Fast for prototyping, but ships ~350 KB of JavaScript and cannot be purged.

**Option C — CDN Play in production:**  
Rejected. The Tailwind team explicitly states the CDN script is not intended for production. It triggers a CSP violation if `script-src` is restricted, and the 350 KB runtime impacts Core Web Vitals.

**Tailwind build integration with MSBuild:**  
Adding an MSBuild `<Target>` to the `.csproj` that calls `npx tailwindcss` on `BeforeBuild` makes the project self-contained but requires Node.js to be present on every developer machine and the CI agent. This is acceptable and is the recommended approach.

Example `.csproj` target (for implementers):

```xml
<Target Name="BuildTailwindCSS" BeforeTargets="Build">
  <Exec Command="npx tailwindcss -i Themes/Tailwind/Assets/tailwind.input.css -o wwwroot/themes/tailwind/tailwind.css --minify"
        WorkingDirectory="$(ProjectDir)"
        ContinueOnError="true" />
</Target>
```

The `ContinueOnError="true"` allows builds to succeed even if Node.js is absent (the previously compiled file is used). A warning is emitted; `dotnet run` still works.

## Consequences

- **Positive:** Production CSS is purged — typically 5–20 KB for this application's class surface, compared to the full Tailwind stylesheet.
- **Positive:** No JavaScript runtime injected into the page by Tailwind itself; the Alpine.js bundle is the only runtime addition.
- **Positive:** Content Security Policy can be set to disallow `unsafe-inline` scripts without any Tailwind CDN Play `<script>` to exempt.
- **Negative:** Developers must have Node.js and npm available. This is a soft requirement (CI pipeline can compile; dev machines can use the previously compiled file).
- **Negative:** The compiled `tailwind.css` must be regenerated whenever new Tailwind utility classes are added to `.cshtml` views. The MSBuild target handles this automatically on each build.
- **Neutral:** The input file `Themes/Tailwind/Assets/tailwind.input.css` (with `@import "tailwindcss"` and any custom theme tokens) is the canonical source of truth for the design tokens. This file is committed to source control; the compiled `tailwind.css` output may be committed or `.gitignore`d depending on team preference.
